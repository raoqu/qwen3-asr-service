# Qwen3-ASR Service API 文档（v1，兼容版本）

**中文** | [English](v1_EN.md)

v1 为兼容旧客户端保留。**新接入请使用 [v2 文档](v2.md)（默认版本）**。

## 与 v2 的关系

v1 与 v2 的离线批处理、健康检查、能力查询接口**共用同一实现**，行为完全一致，仅路径前缀不同（`/v1` vs `/v2`）。各接口的参数、响应结构、错误码均以 [v2 文档](v2.md) 为准，把路径中的 `/v2` 替换为 `/v1` 即可：

| 接口 | v1 路径 | 说明 |
|------|---------|------|
| 提交 ASR 任务 | `POST /v1/asr` | 同 v2 |
| 获取任务列表 | `GET /v1/tasks` | 同 v2（含 `status` / `history` / `limit` 参数） |
| 查询任务详情 | `GET /v1/tasks/{task_id}` | 同 v2 |
| 取消 / 删除任务 | `DELETE /v1/tasks/{task_id}` | 同 v2（含历史任务删除 `deleted` 语义） |
| 健康检查 | `GET /v1/health` | 同 v2 |
| 能力查询 | `GET /v1/capabilities` | 同 v2 |

认证方式相同：配置 API 密钥后离线接口需 `Authorization: Bearer <key>`。

## v1 独有：已过时的任务查询别名

```
GET /v1/asr/{task_id}
```

`GET /v1/tasks/{task_id}` 的历史别名，行为完全相同。**已标记过时（deprecated），仅 v1 保留**，v2 不提供。仅供旧客户端过渡使用，请尽快迁移。

## v1 不提供的能力

- **实时转写**：`WS /v2/asr/stream` 仅在 v2 命名空间提供（统一端点，无 v1 版本），见 [v2 文档 · 实时转写](v2/transcription.md#实时转写)。
- **说话人管理**：`/v2/speakers*`（声纹库登记/识别/管理）仅在 v2 命名空间提供，见 [v2 文档 · 说话人管理](v2/speakers.md#声纹库接口)。注意因共用实现，开启说话人分离/声纹识别后，`POST /v1/asr` 的 `identify_speakers` 参数与离线结果中的 `speaker` / `speaker_name` / `speakers` 字段在 v1 同步生效（additive，见下）。

## 版本演进约定

- 协议变更全部为 additive（新增可选字段/参数），旧客户端忽略未知字段即可，不会被破坏；因 v1/v2 共用实现，此类变更在 v1 同步生效（如任务持久化新增的 `history` 参数与 `wav_name` 字段、说话人分离新增的 `segments[].speaker` 等字段）。
- 全新能力（如实时转写端点、说话人管理接口）只落在 v2 命名空间，v1 不再扩充端点。
