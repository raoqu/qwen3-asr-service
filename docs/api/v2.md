# Qwen3-ASR Service API 文档（v2，默认版本）

**中文** | [English](v2_EN.md)

所有接口以 `/v2` 为前缀，默认地址 `http://127.0.0.1:8765`。

v1 为兼容旧客户端保留，离线接口与 v2 完全相同（仅前缀不同），见 [v1 文档](v1.md)。

> 服务运行时可访问 `http://127.0.0.1:8765/docs` 打开 Swagger UI（FastAPI 自动生成的交互式接口调试页）。注意：该链接仅在服务运行时有效，在 GitHub 上浏览本文档无法跳转；且 Swagger 页面静态资源从公网 CDN 加载，离线环境下无法显示。

## 文档导航

API v2 按功能拆分为四个子文档：

| 子文档 | 内容 |
|--------|------|
| [基础接口](v2/basics.md) | 认证、服务入口 `GET /`、健康检查 `GET /v2/health`、能力查询 `GET /v2/capabilities` |
| [转写](v2/transcription.md) | 离线批处理提交 `POST /v2/asr`、实时转写 `WS /v2/asr/stream` |
| [任务管理](v2/tasks.md) | 任务列表 / 详情 / 取消删除、转写结果结构、任务持久化对 API 的影响 |
| [说话人管理](v2/speakers.md) | 说话人分离与声纹识别、声纹库接口 `/v2/speakers*` |

> 已接入 OpenAI / 阿里云 DashScope 生态的客户端，可改 base_url 直接对接 [兼容接口](compat.md)（OpenAI / DashScope drop-in，离线 + 实时）。

## 认证速览

服务配置了 API 密钥时，**离线批处理接口**要求携带 Bearer Token，否则返回 `401`；`GET /health`、`GET /capabilities` 免认证；实时 WebSocket 与说话人管理接口另有约定。完整规则见[基础接口 · 认证](v2/basics.md#认证)。

```bash
curl -H "Authorization: Bearer sk-your-key-here" http://127.0.0.1:8765/v2/tasks
```

## 典型调用流程

- **离线转写**：[`POST /v2/asr`](v2/transcription.md#提交-asr-任务) 提交音频拿到 `task_id` → 轮询[`GET /v2/tasks/{task_id}`](v2/tasks.md#查询任务详情) 获取识别结果。
- **实时转写**：先 [`GET /v2/capabilities`](v2/basics.md#能力查询) 预检 `stream.enabled` → 建立 [`WS /v2/asr/stream`](v2/transcription.md#实时转写) 推送 PCM 音频帧、逐句接收 `final`。
- **说话人**：转写时带 `identify_speakers=true` 联动[声纹库](v2/speakers.md#说话人分离与声纹识别)，登记 / 改名见[声纹库接口](v2/speakers.md#声纹库接口)。
